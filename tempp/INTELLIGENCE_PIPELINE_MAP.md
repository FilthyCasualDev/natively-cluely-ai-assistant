# INTELLIGENCE_PIPELINE_MAP.md

## End-to-End Intelligence Flow

### Current Flow (Before Fixes)

```
USER INPUT
    │
    ├── Resume upload ──────────────────────────────────────────────────────────┐
    │   ProfileIntelligenceSettings.tsx ─► profile:upload-resume IPC         │
    │   ─► KnowledgeOrchestrator.ingestDocument()                          │
    │       1. extractDocumentText() → raw text                            │
    │       2. extractStructuredData() → LLM → StructuredResume JSON    │
    │       3. processResume() → normalized StructuredResume              │
    │       4. chunkAndEmbedDocument() → nodes + embeddings               │
    │       5. saveDocument() → KnowledgeDatabaseManager (SQLite)         │
    │       6. AOT pipeline (JD only): gap analysis, negotiation script │
    │   ─► refreshCache() → activeResume / activeJD in-memory cache       │
    │                                                                       │
    ├── JD upload ───────────────────────────────────────────────────────────┤
    │   Same pipeline as resume, type=DocType.JD                          │
    │   AOT pipeline runs after JD ingest                                  │
    │                                                                       │
    ├── Custom Context (notes) ─────────────────────────────────────────────┤
    │   ProfileIntelligenceSettings ─► profile:save-notes IPC              │
    │   ─► DatabaseManager.saveNotes() + LLMHelper.setCustomNotes()       │
    │   ⚠️ LLMHelper.customNotes is SEPARATE from KnowledgeOrchestrator.customNotes │
    │   KnowledgeOrchestrator.setCustomNotes() NOT wired to LLMHelper!       │
    │                                                                       │
    ├── AI Persona ────────────────────────────────────────────────────────┤
    │   ProfileIntelligenceSettings ─► profile:save-persona IPC            │
    │   ─► DatabaseManager.savePersona() + LLMHelper.setPersonaPrompt()   │
    │   ⚠️ Persona goes to LLMHelper.personaPrompt, NOT to orchestrator   │
    │                                                                       │
    ├── Negotiation settings ──────────────────────────────────────────────┤
    │   Generated by NegotiationEngine during AOT pipeline (JD ingest)      │
    │   Stored in DB via KnowledgeDatabaseManager                          │
    │                                                                       │
    └── Persona Engine toggle ──────────────────────────────────────────────┤
        ProfileIntelligenceSettings ─► profile:set-mode IPC                 │
        ─► orchestrator.setKnowledgeMode(true) + SettingsManager        │
        ─► knowledgeModeActive flag set in orchestrator                  │
```

### Frontend → Backend Chat Flow

```
MeetingChatOverlay / GlobalChatOverlay / NativelyInterface
    │
    ├── buildContextString() ─► assembles transcript + meeting summary
    │
    ├── streamGeminiChat(message, context, systemPrompt, options)
    │   ⚠️ options.ignoreKnowledgeMode was hardcoded to TRUE in overlays
    │   ─► IPC: gemini-chat-stream
    │
    └── IPC electron main process
            ─► LLMHelper._streamChatInner()
```

### LLMHelper._streamChatInner() — Knowledge Intercept

```
_streamChatInner(message, context, systemPromptOverride?, ignoreKnowledgeMode)
    │
    ├── 1. shouldRunKnowledge = !ignoreKnowledgeMode && !groqFastTextMode
    │                              && knowledgeOrchestrator?.isKnowledgeMode()
    │   ⚠️ ignoreKnowledgeMode=TRUE from overlays → shouldRunKnowledge=FALSE
    │
    └── 2. IF shouldRunKnowledge:
            │
            ├── knowledgeOrchestrator.feedForDepthScoring(message)
            │
            ├── knowledgeResult = processQuestion(message)
            │   │
            │   ├── classifyIntent() ─► INTRO, PROFILE_DETAIL, NEGOTIATION, etc.
            │   │   ├── INTRO_PATTERNS checked first
            │   │   └── IDENTITY_DIRECT_PATTERNS checked (NEW: added "what is my name")
            │   │
            │   ├── CANDIDATE_FRAMING_REGEX ─► /\b(you|your|...|me|my|mine)\b/i (NEW)
            │   │
            │   ├── isGenericKnowledgeQuestion() ─► false for candidate questions
            │   │
            │   └── IF candidate-directed:
            │       ├── getRelevantNodes() via hybrid search
            │       ├── assemblePromptContext() ─► identity header + contextBlock
            │       │   ├── isBareGreeting() ─► greeting shortcut
            │       │   ├── isIntroQuestion() ─► intro shortcut (NEW: includes IDENTITY_DIRECT)
            │       │   ├── buildIdentityHeader() ─► "You generate speech for NAME..."
            │       │   └── buildKnowledgeSystemPrompt() ─► <knowledge_engine_rules>
            │       │
            │       ├── NEGOTIATION: generateLiveCoachingResponse() or salary context
            │       ├── COMPANY_RESEARCH: dossier context from AOT/cache
            │       └── returns PromptAssemblyResult { systemPromptInjection, contextBlock }
            │
            ├── IF knowledgeResult && isPremiumKnowledgeInterceptAllowed():
            │   ├── negotiation coaching → handler (bypass stream)
            │   ├── intro shortcut → yield introResponse (bypass stream)
            │   ├── systemPromptOverride = CORE_IDENTITY + knowledgeSystemPrompt
            │   └── context = knowledgeResult.contextBlock + context
            │
            └── IF knowledgeResult is null (generic/general question):
                └── Falls through to normal LLM path — NO identity injection
                ⚠️ THIS WAS THE BUG: "what is my name?" returned null → no identity
```

### Mode Context Injection (AFTER knowledge intercept)

```
IF !shouldSkipModeInjection:
    ├── ModesManager.getActiveModeSystemPromptSuffix() → mode rules
    └── ModesManager.buildRetrievedActiveModeContextBlock()
        ├── modeContext.customContext → <active_mode_custom_instructions>
        ├── modeContext.referenceFiles → <reference_file> (MAX 40K chars total)
        └── ⚠️ PREMIUM_INTERCEPT_INCOMPATIBLE_TEMPLATES blocks intercept for
            technical-interview, team-meet, lecture modes
```

### Provider Routing

```
├── Multimodal (images) ─► streamVisionWithFallback()
├── Groq fast text ─► Groq direct / Codex CLI
├── Cloud providers ─► systemPromptOverride + cloudCombinedContext (= context, NO persona)
└── Local/Ollama ─► systemPromptOverride + combinedContext (= personaContext + context)
    ⚠️ personaPrompt only reaches LOCAL/OLLAMA path, NOT cloud providers!
```

## Points Where Context Can Be Dropped

| # | Point | Issue | Status |
|---|-------|-------|--------|
| 1 | `ignoreKnowledgeMode: true` in overlays | Hardcoded flag blocked all knowledge path | ✅ FIXED |
| 2 | `isKnowledgeMode()` = false when no resume | Pipeline only active with valid resume | Working as designed |
| 3 | `INTRO_PATTERNS` missing "my name" patterns | "what is my name?" not recognized as intro | ✅ FIXED |
| 4 | `CANDIDATE_FRAMING_REGEX` missing "my/mine" | "my name" not recognized as candidate question | ✅ FIXED |
| 5 | `processQuestion` returns null for GENERAL intent | No identity injection for generic questions | ✅ FIXED (IDENTITY_DIRECT_PATTERNS) |
| 6 | `personaPrompt` not sent to cloud providers | Cloud path uses `cloudCombinedContext = context` | Known limitation |
| 7 | `customNotes` not wired between LLMHelper and orchestrator | Two separate customNotes fields | Design issue |
| 8 | `PREMIUM_INTERCEPT_INCOMPATIBLE_TEMPLATES` blocks intercept | technical-interview/team-meet/lecture modes | Design choice |

## Corrected Flow After Fixes

```
User uploads resume
    │
    ├── ingestDocument() ─► structured_data in DB + activeResume in memory
    │
    └── User enables Persona Engine toggle
            ─► profile:set-mode(true) ─► orchestrator.setKnowledgeMode(true)

User asks "what is my name?" in meeting overlay
    │
    ├── streamGeminiChat(question, ..., { skipSystemPrompt: true })
    │   NOTE: ignoreKnowledgeMode no longer hardcoded to true
    │
    ├── LLMHelper._streamChatInner()
    │   └── shouldRunKnowledge = true (knowledgeMode enabled, no override flag)
    │       │
    │       ├── processQuestion("what is my name?")
    │       │   ├── classifyIntent("what is my name?")
    │       │   │   ├── isIntroQuestion() → true (matches IDENTITY_DIRECT_PATTERNS)
    │       │   │   └── returns IntentType.INTRO
    │       │   │
    │       │   ├── "my" in CANDIDATE_REF_REGEX → hasCandidateFraming = true
    │       │   │
    │       │   ├── NOT isGenericKnowledgeQuestion() → false (identity direct)
    │       │   │
    │       │   ├── assemblePromptContext():
    │       │   │   ├── isIntroQuestion() → true
    │       │   │   ├── generateJitIntro():
    │       │   │   │   └── "I'm Aarav Menon, a backend engineer at Stripe..."
    │       │   │   └── returns { isIntroQuestion: true, introResponse: "..." }
    │       │   │
    │       │   └── returns PromptAssemblyResult { introResponse: "I'm Aarav..." }
    │       │
    │       ├── knowledgeResult.introResponse exists → yield "I'm Aarav Menon..." → return
    │       │
    │       └── LLM never called → identity returned directly
```

## Context Priority Order

1. **Explicit current user instruction** — the user's current question
2. **Negotiation live coaching** — live negotiation handler (bypasses LLM)
3. **JIT intro response** — generateJitIntro() output (bypasses LLM)
4. **Knowledge system prompt** — CORE_IDENTITY + knowledge_engine_rules
5. **Active JD context** — salary intelligence, gap analysis, mock questions, culture mappings
6. **Resume context nodes** — hybrid search results from resume/JD chunks
7. **Mode custom instructions** — MODE_POLICY trust level
8. **Mode reference files** — UNTRUSTED_REFERENCE trust level
9. **Persona (free-text)** — USER_PROVIDED PERSONA CONTEXT (local/Ollama only)
10. **Custom notes** — user_context block (orchestrator path only)
11. **Meeting transcript** — UNTRUSTED_TRANSCRIPT trust level
12. **Screen context** — UNTRUSTED_SCREEN trust level
13. **Meeting history** — UNTRUSTED_MEETING_HISTORY trust level
